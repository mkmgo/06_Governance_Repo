VS Code Organisation

Organizing projects, folders, and files in VS Code relies on establishing a clear, consistent folder structure and leveraging VS Code's Workspace features.

A well-organized structure enhances collaboration, scalability, and maintainability. While the best structure depends on your project type (e.g., frontend, backend), here are general principles and common conventions:

1. Core Principles

  • Consistency: Use a consistent naming convention (e.g., kebab-case for files, camelCase for variables, PascalCase for components) across your project.
  • Modularity: Break your code into small, reusable units.
  • Separation of Concerns: Group files based on their function (e.g., UI, business logic, data access).

2. Common Folder Conventions

Most modern projects follow a structure rooted in a main source folder.

Folder Name Purpose Example Contents
src/ or app/ Contains the main application source code. All your core logic and application files.
assets/ For static resources. Images, fonts, icons, videos.
components/ For reusable UI or logical components. Individual buttons, navigation bars, or data modules.
services/ or utils/ For functions that handle business logic, API calls, or common utilities. API wrappers, date formatting helpers.
styles/ For global CSS/Sass or theme files. global.css, _variables.scss.
tests/ Contains all unit and integration tests. Files like login.test.js.
docs/ For project documentation. Markdown files, specifications.
.vscode/ For VS Code-specific configuration. settings.json, tasks.json, launch.json.

3. Organization Styles

  • By Type (Smaller Projects): Files are grouped strictly by their technical type (e.g., all components in components/, all services in services/).
  • By Feature (Larger Projects): Files related to a specific feature are grouped into a single folder (e.g., an authentication/ folder might contain its components, services, and tests). This is often better for scaling.

üíª Managing Projects with VS Code Workspaces

VS Code uses a concept called a Workspace to manage one or more project folders.

1. Single-Folder Workspace

  • This is the simplest setup: you open a single folder, which becomes the root of your workspace. This is best for a single repository or small, self-contained project.
  • How to Open: Go to File > Open Folder... and select your project's root folder.

2. Multi-root Workspace

  • This is ideal for larger projects involving multiple related repositories, packages, or microservices (a monorepo structure). It allows you to manage several independent root folders in a single VS Code window.
  • How to Create:
    1. Go to File > Add Folder to Workspace... to add additional folders to your current window.
    2. Go to File > Save Workspace As... and save a .code-workspace file. This file stores the configuration, including which folders are open and their relative paths, allowing you to easily reopen the entire setup later.
  • Benefits: You can search across all projects, manage one set of global extensions, and use a unified terminal.

3. Workspace Settings

You can define settings specific to your project or workspace that override your global User settings. These are saved in a .vscode/settings.json file (for single-folder projects) or within the saved .code-workspace file.

  • To Open: Use the Command Palette (Ctrl+Shift+P) and search for Preferences: Open Workspace Settings.

üõ†Ô∏è VS Code Productivity Tools

  • Extensions: Extensions like Project Manager can help you quickly switch between different saved workspaces or projects.
  • File Explorer: Use the built-in Explorer (Ctrl+Shift+E) to quickly navigate your structure.
  • Quick Open: Press Ctrl+P (or Cmd+P on Mac) and start typing to instantly search for and open any file in your workspace.
  • Breadcrumbs: Enable View > Show Breadcrumbs to see and navigate the file's hierarchy at the top of the editor.
